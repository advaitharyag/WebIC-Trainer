# Contributing to Web IC Trainer

Thanks for your interest in contributing.

## Development Setup

1. Fork and clone the repository.
2. Open `ic-trainer-simulator.html` in a browser.
3. Make focused, testable changes.

## Contribution Guidelines

- Keep PRs small and focused.
- Preserve current layout behavior unless the PR is specifically a layout change.
- For simulation logic changes, include test notes in the PR description.
- Update docs for any user-visible behavior changes.

## Pull Request Checklist

- [ ] Code runs in browser without console errors.
- [ ] Existing behaviors are not regressed.
- [ ] README/docs updated if needed.
- [ ] Changelog updated under `[Unreleased]`.

## Commit Message Suggestions

- `feat: add ...`
- `fix: resolve ...`
- `docs: update ...`
- `refactor: simplify ...`

## Reporting Bugs

Please include:

- Steps to reproduce
- Expected behavior
- Actual behavior
- Browser and OS details
